# Claude Code Orchestration Harness

Claude Code를 체계적으로 굴리기 위한 오케스트레이션 프레임워크.

코드를 짤 필요 없다. Claude Code한테 말로 시키면 된다.

## 개요

Claude Code는 강력하지만, 그냥 쓰면 이런 문제가 생긴다:

| 문제 | 원인 |
|------|------|
| 세션 바뀌면 맥락 전부 소실 | 이전 결정사항을 기억 못함 |
| 에러 처리 누락, 보안 실수 | 매번 사람이 체크해야 함 |
| 복잡한 작업에서 품질 저하 | 혼자 다 하려다 실수 |
| "뭐 하면 되죠?" 반복 | 진행 상황 추적 안됨 |

이 하네스는 **훅 + 스킬 + 에이전트 + 3대 문서** 구조로 위 문제를 자동 해결한다.

## 구조

```
.claude/
├── settings.json              # 권한 설정
├── hooks/
│   ├── user-prompt-submit.sh  # 프롬프트 사전 분석 + 세션 브리핑
│   ├── post-tool-use.sh       # 자동 코드 체크 + 수정 기록
│   └── shared/                # 맥락노트, 체크리스트, 수정기록 (로컬 상태)
├── skills/
│   ├── INDEX.md               # 스킬 목차 (트리거 키워드 포함)
│   ├── ch01-python-quality.md # 메타 스킬: 코드 품질
│   ├── ch02-skill-activation.md # 활성화 규칙표
│   └── chapters/              # 도메인별 스킬 챕터
└── agents/
    ├── domain-agent.md        # 도메인 에이전트 템플릿
    └── auditor-agent.md       # 평가 에이전트 템플릿
```

## 핵심 컴포넌트

### 1. 훅 (자동 실행)

- **UserPromptSubmit** — 지시가 들어오면 Claude가 읽기 전에 자동 분석. 카테고리 분류, 복잡도 판단, 스킬 챕터 자동 로드, 에이전트 추천.
- **PostToolUse** — 코드 수정 직후 자동 체크. 보안 위험, 에러 처리 누락, 프로젝트 규칙 위반 감지. 오류 수에 따라 자동 분기 (경미→즉시 수정, 심각→전문 에이전트 호출).

### 2. 스킬 매뉴얼 (선택적 로드)

프로젝트 도메인별 기술 가이드. 전부 로드하지 않고 키워드 감지 시 해당 챕터만 자동 주입.

### 3. 에이전트 (전문가 분업)

- **도메인 에이전트** — 기능 영역별 전문가. 자기 영역만 작업.
- **평가 에이전트** — 작성자와 검증자 분리. 코드 품질/기획 일치도를 독립 검증.

### 4. 3대 문서 (맥락 유지)

| 문서 | 역할 |
|------|------|
| 기획서 | 뭘 만들 건지 (읽기 전용) |
| 맥락노트 | 왜 이렇게 결정했는지 |
| 체크리스트 | 뭘 끝냈고 뭐가 남았는지 |

## 사용법

1. `docs/ORCHESTRATION_SETUP_GUIDE.md`를 읽는다
2. `templates/` 안의 파일들을 프로젝트의 `.claude/`에 복사한다
3. 프로젝트에 맞게 스킬 챕터와 에이전트를 커스터마이징한다

**또는** 그냥 Claude Code한테 이렇게 말한다:

> "이 가이드 보고 우리 프로젝트에 맞게 오케스트레이션 세팅해줘"

## 전체 흐름

```
① 지시 내리기
   ↓
② 훅: 분석 + 매뉴얼 자동 전달 + 에이전트 추천
   ↓
③ AI(또는 에이전트)가 작업 수행
   ↓
④ 훅: 수정 기록 자동 저장
   ↓
⑤ 훅: 오류 자동 검사 + 분기
   ↓
⑥ 보고서 출력
   ↓
⑦ 체크리스트 체크 + 맥락노트 업데이트
   ↓
⑧ 다음 작업으로
```
